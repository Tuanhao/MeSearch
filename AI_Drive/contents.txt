In short, we generate and share requirements.txt files to make it easier for other developers to install the correct versions of the required Python libraries (or “packages”) to run the Python code we’ve written.</p><h2 class="im in ap bx bw io ip iq ir is it iu iv iw ix iy iz" id="ff61">Python Packages, and Environment</h2><p class="gi gj ap bx gk b dy ja ea jb gn jc gp jd gr je gt cx" id="e545">Open-source python packages — like <a class="bg fw gu gv gw gx" href="https://www.crummy.com/software/BeautifulSoup/" rel="noopener nofollow" target="_blank">beautifulsoup</a>, or <a class="bg fw gu gv gw gx" href="https://jupyter.readthedocs.io/en/latest/architecture/visual_overview.html" rel="noopener nofollow" target="_blank">jupyter</a>, or any of the other 158,872+ projects on the <a class="bg fw gu gv gw gx" href="https://pypi.org/" rel="noopener nofollow" target="_blank">PyPi</a> index — offer tremendous functionality, way beyond that of the standard Python library. Every time you <code class="hj hk hl hm hn b">$ pip install</code>something, you are expanding your python environment, giving it access to packages that are not part of the <a class="bg fw gu gv gw gx" href="https://docs.python.org/3/library/index.html" rel="noopener nofollow" target="_blank">Python standard library</a>.</p><p class="gi gj ap bx gk b dy gl ea gm gn go gp gq gr gs gt cx" id="3808">If you <code class="hj hk hl hm hn b">$ pip install</code> a bunch of stuff outside of a <strong class="gk jk">virtual environment </strong>(more on this later), then you are adding to your “base” or “root” or “system” python environment. That is fine and good and totally valid for many sandbox-y purposes…</p><p class="gi gj ap bx gk b dy gl ea gm gn go gp gq gr gs gt cx" id="9526">However, working exclusively in your base environment will inevitably cause headaches later, when you try to show the code you’ve built to <strong class="gk jk">other human beings</strong>. In the future, even after many new versions of each package (including python itself) have come and gone, we will be able to re-hydrate this little retro bubble world, and re-populate it with the correct versions of all the packages needed to make our old code happy.</p><p class="gi gj ap bx gk b dy gl ea gm gn go gp gq gr gs gt cx" id="178d">The Python 3 standard library has built-in venv creation capabilities, but I don’t feel like talking about that.
